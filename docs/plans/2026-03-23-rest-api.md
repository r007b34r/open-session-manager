# REST API Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 补齐 `API-01`，让 OSM 在本地提供一个可测试的只读 REST API，复用现有查询层输出会话列表、搜索结果和详情。

**Architecture:** 新增独立的 `api_server` 模块，不碰现有 Tauri command 和受保护的会话控制实现。CLI 增加 `serve` 子命令，使用共享 `DashboardSnapshot` 加载逻辑和 `commands::query`，对外暴露 `/health`、`/api/v1/sessions`、`/api/v1/sessions/search`、`/api/v1/sessions/:id`、`/view`、`/expand` 六类 GET 接口。

**Tech Stack:** Rust, axum, tokio, cargo test

---

### Task 1: 先写 REST API 红测

**Files:**
- Create: `src-tauri/tests/http_api.rs`

**Step 1: Write the failing tests**

Cover:
- `GET /health` 返回稳定 JSON 健康状态
- `GET /api/v1/sessions` 支持 `assistant/limit/offset/sortBy/descending`
- `GET /api/v1/sessions/search` 支持 `query` 与共享排序
- `GET /api/v1/sessions/{sessionId}`、`/view`、`/expand` 返回稳定 payload

**Step 2: Run tests to verify they fail**

Run:
- `cargo test --test http_api --manifest-path src-tauri/Cargo.toml`

**Step 3: Commit**

